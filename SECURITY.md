# Security Policy

## Supported Versions

Security updates are applied to the latest `main` branch.

| Version | Supported |
|---|---|
| main | Yes |
| older branches | No |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue first.

Report privately to:

- umesh.code1@gmail.com

Include:

- Description of the vulnerability
- Impact and affected components
- Reproduction steps or proof-of-concept
- Suggested mitigation (if available)

## Response Process

- Initial acknowledgement target: within 72 hours
- Triage and severity assessment
- Fix development and validation
- Coordinated disclosure after patch is available
